Puppet 3.2 - Experimental Parser

Alex W
2013-06-12

The Future's Bright

  • Puppet is a widely used configuration management tool.
  • Version 3.2 contains an additional parser
  • The reimplementation of the parser is an attempt to expand the scope of the language, and fix "warts"
  • Move from a non-expression-based grammar to an EBG
  • Fully backwards compatible.
  • Either add parser = future to the puppetmaster config, or run puppet apply --parser future

Lambda Functions

  • Almost all sensible modern languages have lambda functions as part of the syntax.
  • Puppet adopts the ruby-like syntax for these functions
  • Example 
    $things = ["foo", "bar", "baz"]
$things.each |$thing| { notice $thing }
  • Available are each,collect, reduce, and other standard functional operations.

Other features

  • Arrays and hashes are now allowed “everywhere”
    • notice(["foo", "bar", "baz"][2]) // ... Scope(Class[main]): baz
  • Array operations too!
    • ["foo","bar"] + ["baz","zap"] // ... ["foo","bar","baz","zap"]
{"foo" => 1} + {"bar" => 2}   // ... {"foo" => 1, "bar" => 2}
  • Function interpolation
    • notice "RN: ${fqdn_rand(30)}" // .. Scope(Class[main]): RN: 9

Closing Remarks

  • Improved error reporting.
  • Iterate new features, but not break backwards compatibility
  • More "language-like", which some people like
  • Interesting to play with before it becomes integrated
